2
Push Code to GitHub
Render deploys directly from a GitHub repository
1️⃣

Create a GitHub account (if you don't have one)

Go to github.com → Sign Up (free). Verify your email.

2️⃣

Create a new repository

Click + New → Name it eipl-inventory → Select Private → Click Create repository.

3️⃣

Upload your files

On the new repo page, click "uploading an existing file" → Drag and drop all your project files → Click Commit changes.

✅ No Git installation needed — GitHub's web interface works fine for this.
3
Set Up Supabase PostgreSQL Database
Free managed PostgreSQL — replaces your local inventory.db
1️⃣

Create a Supabase account

Go to supabase.com → Sign Up with GitHub (easiest) → Free tier is enough.

2️⃣

Create a new project

Click New Project → Enter a name like eipl-inventory → Set a strong database password (save this!) → Choose region: Southeast Asia (Singapore) for best speed from India → Click Create Project.

⚠️ Save your database password — you cannot recover it later. Write it down now.
3️⃣

Get your connection string

In your Supabase project → Go to Settings (⚙️ in left sidebar) → Database → Scroll to "Connection string" → Select URI tab → Copy the string.

It will look like:

postgresql://postgres:[YOUR-PASSWORD]@db.xxxxxxxxxxxx.supabase.co:5432/postgres

Replace [YOUR-PASSWORD] with the password you set above. Keep this string safe — it is your database key.

4
Deploy on Render
Connect GitHub repo → set DATABASE_URL → deploy
1️⃣

Create a Render account

Go to render.com → Sign Up with GitHub → This links your GitHub repos automatically.

2️⃣

Create a new Web Service

Click New + → Web Service → Connect your eipl-inventory GitHub repo → Click Connect.

3️⃣

Configure the service settings

FieldValue to Enter
Nameeipl-inventory
RegionSingapore (closest to India)
Branchmain

Start Commanduvicorn app:app --host 0.0.0.0 --port $PORT

4️⃣

Add your DATABASE_URL environment variable

Scroll down to Environment Variables section → Click Add Environment Variable:

KeyValue
DATABASE_URLPaste your full Supabase PostgreSQL connection string here
⚠️ Critical: Your database.py already reads this variable. Do NOT hardcode the database URL in any file.
5️⃣

Click "Create Web Service"

Render will now install packages and start your app. The first deploy takes about 2–4 minutes. Watch the live log for any errors.

✅ Your app automatically runs models.Base.metadata.create_all() on startup — this creates all tables in Supabase automatically. No manual SQL needed!

6
Important Limits & Tips (Free Tier)
Know what to expect on the free plan
😴

App "sleeps" after 15 minutes of inactivity

On the free Render plan, if no one visits the app for 15 minutes, it spins down. The next visitor will wait ~30–60 seconds for it to wake up. This is normal.

💡 To avoid this, you can use a free service like UptimeRobot (uptimerobot.com) to ping your URL every 10 minutes and keep it awake.
🗄️

Supabase free tier limits

⚠️ Supabase pauses your free project if there's no activity for 7 days. Log into Supabase and click Restore if your app can't connect.
🔄

How to update your app in future

Edit your code locally → Go to GitHub → Upload the changed files (or drag & drop to replace). Render auto-detects the change and re-deploys within 2 minutes.

🔒

Security note

Your app uses SHA-256 password hashing which is good. Consider adding a SECRET_KEY environment variable for session security in future upgrades.
